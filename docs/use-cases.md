# Use-Case модель Lamoda

## Основные акторы

- `Гость` - неавторизованный посетитель сайта.
- `Покупатель` - авторизованный пользователь, оформляющий заказ.
- `Платежная система` - внешний сервис подтверждения оплаты.
- `Служба доставки` - внешний сервис выбора и отслеживания доставки.

## Прецеденты использования

| ID | Прецедент | Краткое описание | Автоматизация |
|---|---|---|---|
| UC-01 | Просмотр главной страницы | Пользователь открывает сайт и видит основные разделы каталога | `HomePageSmokeTest` |
| UC-02 | Поиск товара | Пользователь вводит поисковый запрос и получает список товаров | `SearchFlowTest` |
| UC-03 | Просмотр каталога | Пользователь открывает категорию каталога | `CatalogPageTest` |
| UC-04 | Фильтрация и сортировка | Пользователь видит элементы управления выдачей | `CatalogPageTest` |
| UC-05 | Просмотр карточки товара | Пользователь открывает страницу товара и изучает характеристики | `ProductPageTest` |
| UC-06 | Просмотр корзины | Пользователь открывает корзину | `CartPageTest` |
| UC-07 | Добавление в корзину | Пользователь выбирает размер и добавляет товар | Частично отражено в `ProductPageTest`, требует стабильного тестового товара |
| UC-08 | Авторизация | Пользователь входит перед оформлением заказа | Не автоматизировано в текущем наборе |
| UC-09 | Оформление заказа | Пользователь подтверждает доставку и оплату | Не автоматизировано из-за внешних интеграций и риска влияния на боевую среду |

## Почему выбран именно такой набор автотестов

- Тесты покрывают базовые и безопасные для production-площадки сценарии.
- Все локаторы построены на `XPath`, без привязки к `id`.
- Набор подходит для кроссбраузерного smoke-прогона в `Chrome` и `Firefox`.
- Для запуска против реального `lamoda.ru` нужно окружение, в котором не срабатывает anti-bot challenge `Qrator`.
